Naming (English UI / store)

The product name in English is AI Edits. The backend field remains profiles.ai_credits (each unit = one AI Edit).

Consumable products (App Store / Google Play / RevenueCat)

  • cvgen.pro_10edits — +10 pack
  • cvgen.pro_30edits — +30 pack

Optional app overrides: EXPO_PUBLIC_AI_EDIT_PACK_PRODUCT_ID_10 and EXPO_PUBLIC_AI_EDIT_PACK_PRODUCT_ID_30.

Offering credits_offering (RevenueCat Paywalls)

A dedicated offering (e.g. identifier credits_offering) contains both packages. The app presents this paywall when the user is already Pro or Pro+ and has no AI Edits left.

Build-time env: EXPO_PUBLIC_REVENUECAT_CREDITS_OFFERING_ID (default in code: credits_offering).

The subscription paywall (upgrade to Pro) uses the project’s default/current subscription offering — not this credits offering.

Tests / admin (Supabase)

The CVGen repo’s admin-credit-pack Edge Function can add balance to profiles.ai_credits in test environments. Requires a server secret; never ship an admin token in the app.

Deploy and curl examples live in the app repo under docs/GITHUB_PAGES.md.
