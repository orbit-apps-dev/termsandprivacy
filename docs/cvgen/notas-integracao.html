Nome na interface e nas lojas

Em inglês o produto é referido como AI Edits. No app em português do Brasil usamos melhoria(s) com IA. O saldo numérico no backend continua em profiles.ai_credits (cada unidade = 1 melhoria).

Produtos consumíveis (App Store / Google Play / RevenueCat)

  • cvgen.pro_10edits — pacote +10 melhorias
  • cvgen.pro_30edits — pacote +30 melhorias

Override opcional no app: EXPO_PUBLIC_AI_EDIT_PACK_PRODUCT_ID_10 e EXPO_PUBLIC_AI_EDIT_PACK_PRODUCT_ID_30.

Offering credits_offering (RevenueCat Paywalls)

Foi criado um offering separado (ex.: identifier credits_offering) contendo os dois pacotes acima. O app abre esse paywall quando o usuário já é Pro ou Pro+ e fica sem melhorias com IA (revisão completa, adaptar à vaga, etc.).

Variável de ambiente no build: EXPO_PUBLIC_REVENUECAT_CREDITS_OFFERING_ID (padrão no código: credits_offering).

O paywall de assinatura (upgrade para Pro) continua sendo o offering atual/padrão do projeto — não confundir com o de créditos.

Testes / admin (Supabase)

Para creditar saldo manualmente em ambiente de teste, a Edge Function admin-credit-pack no repositório do CVGen atualiza profiles.ai_credits. Exige segredo no servidor; não exponha token de admin no app.

Detalhes de deploy e exemplo curl estão no repositório do app em docs/GITHUB_PAGES.md.
